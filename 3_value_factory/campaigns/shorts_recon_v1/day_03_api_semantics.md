# Сценарий: Твой API тебе врет

**Hook:** Статус 200 OK, а в теле ответа `{ error: "User not found" }`? За такое нужно лишать лицензии разработчика.

**Body:** 
1. HTTP статус-коды придумали не для красоты. Это семантический контракт.
2. Если ты возвращаешь 200 на ошибку — ты ломаешь мониторинг, кэширование и голову фронтенд-разработчику.
3. Твой API должен говорить правду даже когда ему больно.

**Insight:** API Design — это про доверие между системами. Хаос в кодах — это хаос в архитектуре.

**CTV:** Хочешь строить API, которые понимают и люди, и машины? Подписывайся.
