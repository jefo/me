# Проект: Cloud Functions SDK (Подобный Supabase)

Инструмент, выросший из необходимости дешево и надежно деплоить вебхуки для Telegram-ботов и Paywall-систем.

## Проблема, которую решает:
- Serverless Framework разочаровал отсутствием поддержки российских облачных провайдеров.
- Необходимость платить только за вызовы (Cloud Functions), а не за простаивающий хостинг серверов.
- Привязка к одному вендору (Vendor Lock-in) при запуске MVP.

## Как это работает:
- **Абстракция над облаками:** Разработчик пишет код один раз, а деплоить может в любое облако под капотом.
- **Бесшовный переезд:** Можно запустить MVP на Яндекс Облаке, а в релизе переехать на VK Cloud или Ktel (Selectel) без переписывания инфраструктурного кода.
- Построен на базе моего [DDD Фреймворка](./ddd-framework.md).
